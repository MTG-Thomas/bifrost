# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately:

- Open a GitHub security advisory (preferred)
- Or email the maintainer directly if available

Please do not open public issues for security vulnerabilities.

## Scope

This project accepts reports for:
- Authentication and authorization flaws
- Secrets exposure
- Injection vulnerabilities
- Dependency vulnerabilities

## Response

We aim to acknowledge reports within 72 hours and provide a remediation plan where applicable.
